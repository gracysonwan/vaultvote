About VaultVote

The Basics of
Campus Digital
Voting

A cryptographically secure e-voting platform demonstrating real-world cybersecurity. Every vote hashed, every identity verified, every action auditable.

Zero-trust architecture: votes and voter identities live in separate, anonymized database tables. Your name never touches your ballot.

  • ✓
    PBKDF2-SHA256 password hashing (310,000 iterations)
  • ✓
    Anonymized ballots — identity & vote in separate tables
  • ✓
    Double-vote prevention — cryptographic session + DB flag
  • ✓
    Blockchain chain hash — tamper-evident audit trail
  • ✓
    Public audit log — verify without losing privacy
# PBKDF2 Password Hashing
import hashlib, secrets

def hash_password(pwd, salt, uid):
  return hashlib.pbkdf2_hmac(
    'sha256',
    (pwd+uid).encode(),
    salt.encode(),
    310_000  # NIST standard
  ).hex()

# SHA-256 Vote Chain
def chain_hash(prev, vote, ts):
  data = f"{prev}:{vote}:{ts}"
  return sha256(data)
The Process

How VaultVote Works

01
Register
Student ID hashed with PBKDF2-SHA256 + unique salt. We store only the hash, never your plain password.
02
Authenticate
Re-hash credentials on login. Constant-time compare prevents timing attacks.
03
Cast Vote
Select candidate. Unique nonce + SHA-256 generates anonymous vote hash. Identity never stored with vote.
04
Chain & Record
Vote hash chained to previous entry. Tamper one entry and every subsequent hash breaks.
05
Verify
Your Vote ID proves your vote exists in the audit log without revealing who you voted for.
